# 🔄 Gleedoc 브랜치 비교 가이드

## 브랜치 현황

### `main` 브랜치 (현재 버전)
```
medical-illustration-portfolio/
├── index.html           # 바닐라 HTML
├── css/styles.css       # 순수 CSS
└── js/
    ├── script.js        # 바닐라 JavaScript
    └── translations.js  # i18n 데이터
```

- **기술 스택**: HTML5 + CSS3 + Vanilla JS
- **번들 크기**: ~50KB
- **초기 로딩**: ⚡ 매우 빠름 (1-2초)
- **애니메이션**: CSS 기반 (기본적)
- **특징**: 가볍고 빠르고 SEO 친화적

---

### `feature/react-redesign` 브랜치 (새 버전)
```
react-app/
├── src/
│   ├── components/      # React 컴포넌트
│   ├── App.jsx         # 메인 앱
│   └── main.jsx        # 엔트리 포인트
├── vite.config.js      # Vite 번들러
└── package.json        # 의존성
```

- **기술 스택**: React 18 + Framer Motion + Tailwind + Vite
- **번들 크기**: ~200KB (압축 시 ~60KB)
- **초기 로딩**: ~3-4초 (React 파싱)
- **애니메이션**: Framer Motion (고급)
- **특징**: 현대적, 상호작용 풍부, 컴포넌트 기반

## 🎯 각 버전의 장단점

### ✅ `main` (바닐라 JS) - 현재 프로덕션

#### 장점
- ⚡ **초경량**: 50KB만으로 완벽한 기능
- 🚀 **빠른 로딩**: LCP < 1초
- 🔍 **SEO 최적**: 검색 엔진이 모든 콘텐츠 인식
- 📱 **모바일 성능**: 저사양 기기도 부드러움
- 🛡️ **간단함**: 의존성 0개 (외부 라이브러리 없음)
- 💰 **호스팅 저렴**: 용량 적어서 대역폭 절약

#### 단점
- 😕 **UI 인터랙션 기본**: 마그네틱 커서, 고급 애니메이션 없음
- 🔧 **상태 관리 복잡**: `currentLang`, 모달 상태 수동 관리
- 🔀 **코드 재사용 어려움**: 컴포넌트화 미흡
- 📈 **확장성 떨어짐**: 기능 추가 시 복잡도 증가
- 🎨 **디자인 고전적**: Bold한 편집사 스타일 구현 어려움

#### 추천 대상
- ✅ SEO 중요한 일반 포트폴리오
- ✅ 빠른 로딩 필수 (모바일 중심)
- ✅ 간단한 구조
- ✅ 저비용 호스팅

---

### ✨ `feature/react-redesign` (React) - 신규 디자인

#### 장점
- 🎨 **현대적 디자인**: Bold Editorial 스타일로 프리미엄 느낌
- ✨ **고급 애니메이션**: Framer Motion으로 우아한 인터랙션
  - 마그네틱 커서
  - Letter-by-letter text reveal
  - Smooth page transitions
- 🧩 **컴포넌트 재사용**: 유지보수 쉬움
- 🤖 **자동 상태 관리**: 언어, 모달 자동 동기화
- 🚀 **빠른 개발**: React 생태계의 강력한 도구
- 💼 **엔터프라이즈급**: 팀 협업에 최적화

#### 단점
- 📦 **무거움**: 200KB+ 번들 (초기 로딩 3-4초)
- 📱 **모바일 성능**: 저사양 기기에서 약간 느림
- 🔍 **SEO 약점**: 클라이언트 사이드 렌더링 (별도 최적화 필요)
- 🧠 **학습곡선**: React 지식 필요
- 🔧 **설정 복잡**: Vite, Tailwind, Framer Motion 설정

#### 추천 대상
- ✅ 디자인/애니메이션 중요한 프리미엄 포트폴리오
- ✅ 브랜드 차별화 필요
- ✅ 장기적 확장성
- ✅ 팀 프로젝트 (유지보수 쉬움)
- ✅ 성능보다 UX/DX 우선
